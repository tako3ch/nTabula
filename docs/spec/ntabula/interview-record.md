# nTabula ヒアリング記録

**作成日**: 2026-03-16
**ヒアリング実施**: kairo-requirements step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

`rev-requirements` で生成した設計文書・実装コードを確認し、不明点・曖昧部分・追加要件を明確化するためのヒアリングを実施しました。

---

## 質問と回答

### Q1: 作業規模の確認

**質問日時**: 2026-03-16
**カテゴリ**: 優先順位
**背景**: 要件定義の詳細度（フル / 軽量 / カスタム）を確認するため

**回答**: フル機能開発（推奨）

**信頼性への影響**:
- フル機能開発を選択したため、ユーザーストーリー・受け入れ基準・非機能要件・エッジケースすべてを作成

---

### Q2: 既存コードベースの詳細分析の要否

**質問日時**: 2026-03-16
**カテゴリ**: 既存設計確認
**背景**: `rev-requirements / rev-tasks` で設計文書が既に存在するが、追加分析が有効か確認するため

**回答**: 必要（Sources/ 以下を再走査し、実装 vs 設計の差分を分析）

**信頼性への影響**:
- コード分析により以下の差分を発見:
  - `TabItem.databaseID` フィールドが未使用であることを確認
  - Notion Integration Token が UserDefaults に平文保存されていることを確認（セキュリティリスク）
  - `page_size=100` で固定・カーソル未使用であることを確認
  - Cmd+W キーバインドが未実装であることを確認

---

### Q3: Notion Integration Token の保存方式

**質問日時**: 2026-03-16
**カテゴリ**: 既存設計確認
**背景**: コード分析で `PersistenceManager.swift` が UserDefaults に平文保存していることを発見。セキュリティ改善要否を確認するため

**回答**: Keychain 移行（推奨）

**信頼性への影響**:
- NFR-101, REQ-403 を 🔵 青信号として追加（Keychain 移行が確定要件）
- prep.md に「Keychain API の実装調査」を推奨タスクとして追加

---

### Q4: 100件以上の Notion データベース・ページへの対応

**質問日時**: 2026-03-16
**カテゴリ**: 既存設計確認
**背景**: コード分析で `NotionListResponse` に `hasMore / nextCursor` が定義されているが未使用であることを発見。大規模ワークスペース対応の要否を確認するため

**回答**: 不要（100件で十分）

**信頼性への影響**:
- REQ-404 を 🔵 青信号として追加（100件上限・ページネーション UI 不要が確定）
- カーソル関連の実装は不要と確定

---

### Q5: Cmd+W（アクティブタブを閉じる）キーバインドの要否

**質問日時**: 2026-03-16
**カテゴリ**: 未定義部分詳細化
**背景**: README に Cmd+T・Cmd+S は記載があるが Cmd+W は記載なし。コード分析でも未実装。標準 macOS UX として必要か確認するため

**回答**: 必要

**信頼性への影響**:
- REQ-105 を 🔵 青信号として追加（Cmd+W でアクティブタブを閉じる）

---

### Q6: テスト戦略（URLSession / UserDefaults のモック化）

**質問日時**: 2026-03-16
**カテゴリ**: 追加要件
**背景**: `docs/spec/ntabula/tests/` にテストコードが存在するが、`NotionService` / `PersistenceManager` への依存注入が未実装のため完全なモックテストができない状態。Unit Test 実装の意向を確認するため

**回答**: Unit Test を実装する

**信頼性への影響**:
- NFR-301, NFR-302, NFR-303 を 🔵 青信号として追加
  - `NotionService` に `init(token:session:)` overload 追加
  - `PersistenceManager` に `init(defaults:)` overload 追加
  - Xcode テストターゲット `nTabulaTests` 追加

---

### Q7: 今後の追加機能の優先度

**質問日時**: 2026-03-16
**カテゴリ**: 優先順位
**背景**: コード分析で実装可能な追加機能候補を複数発見。優先順位を確認するため

**回答**: タブ並び替え（ドラッグ&ドロップ）、Cmd+W キーバインド、Keychain 移行

**信頼性への影響**:
- REQ-305 を 🔵 青信号として追加（タブ D&D 並び替え）
- REQ-105, REQ-403, NFR-101 の優先度を「Must Have」として確定

---

## ヒアリング結果サマリー

### 確認できた事項
- Keychain 移行: 必要（優先度: Must Have）
- ページネーション: 不要（100件で十分）
- Cmd+W: 必要（標準 macOS UX として）
- Unit Test: 実装する（URLSession / UserDefaults 注入含む）
- タブ D&D 並び替え: 追加機能として優先度高

### 追加/変更要件
- REQ-105: Cmd+W でアクティブタブを閉じる（新規追加）
- REQ-305: タブのドラッグ&ドロップ並び替え（新規追加）
- REQ-403: トークン Keychain 保存（UserDefaults から変更）
- REQ-404: ページネーション不要を明示（100件上限・カーソル未使用を許容）
- NFR-101: Keychain 保存（セキュリティ要件として追加）
- NFR-301〜303: テスト可能性要件（新規追加）

### 残課題
- `TabItem.databaseID` フィールドの使用用途の整理（現状未使用）
- ダーク/ライトモードの明示的なテスト方法の確認（自動追従で OK か）
