## エピック: 日本語でストレスなくメモを書く

### ストーリー 1: IME 変換中にテキストが壊れない 🔵

**信頼性**: 🔵 *EditorView.Coordinator.textDidChange L109・AC-005より*

**私は** 日本語でメモするユーザー **として**
**IME 変換候補を選んでいる最中にテキストが消えたり文字化けしたりしないようにしたい**
**そうすることで** 日本語入力に集中できる

**関連要件**: REQ-IME-101, REQ-IME-103

**詳細シナリオ**:
1. エディタに日本語を入力開始（例: "にほんご" とタイプ）
2. macOS IME が変換候補を表示し、テキストに下線マーキングが入る（hasMarkedText() == true）
3. この間、AppState へのコンテンツ更新がスキップされる
4. シンタックスハイライトの再計算もスキップされる
5. ユーザーが変換確定（Return / スペース / 候補クリック）
6. hasMarkedText() == false になり、textDidChange がコンテンツを AppState に反映
7. テキストは完全な形で保持されている

**前提条件**:
- nTabula のエディタにフォーカスがある
- macOS 日本語 IME が有効になっている

**優先度**: Must Have

---

### ストーリー 2: シンタックスハイライトが IME 変換を邪魔しない 🔵

**信頼性**: 🔵 *MarkdownTextStorage.processEditing() L229-230より*

**私は** マークダウン記法を日本語で書くユーザー **として**
**IME 変換中に見出し（`# `）や太字（`**`）の入力が正しく変換されるようにしたい**
**そうすることで** 日本語マークダウン記法をスムーズに入力できる

**関連要件**: REQ-IME-103, REQ-IME-201

**詳細シナリオ**:
1. `# ` に続けて「にほんご」と入力
2. IME 変換候補が表示される
3. MarkdownTextStorage の processEditing() が呼ばれるが、hasMarkedText() == true のためハイライトスキップ
4. IME のマーキング属性（下線）が正しく表示される
5. 変換確定後、シンタックスハイライトが正しく適用される

**優先度**: Must Have

### ストーリー 3: タブ切り替えで IME が妨げない 🔵

**信頼性**: 🔵 *EditorView.updateNSView() L68-74・ユーザヒアリングより*

**私は** 複数タブを使うユーザー **として**
**IME 変換中に別のタブに切り替えたとき、新しいタブのコンテンツが正しく表示されるようにしたい**
**そうすることで** タブ操作をいつでも自由にできる

**関連要件**: REQ-IME-104, REQ-IME-105, NFR-IME-003

**詳細シナリオ**:
1. タブ A で日本語変換中（hasMarkedText() == true）
2. タブ B をクリックして切り替え
3. currentTabID が変わるため updateNSView が textView.string を更新（IME マーキングは破棄）
4. タブ B のコンテンツが正しく表示される
5. 後でタブ A に戻ると、確定前の未確定テキストは消えているが、最後に AppState に保存されたコンテンツは保持される

**前提条件**:
- 2つ以上のタブが存在する

**優先度**: Must Have

### ストーリー 4: IME 変換中でも Cmd+S で保存できる 🔵

**信頼性**: 🔵 *NTTextView.keyDown・ユーザヒアリング 2026-03-16より*

**私は** こまめに Notion に同期したいユーザー **として**
**日本語変換中に Cmd+S を押したら Notion に同期されるようにしたい**
**そうすることで** 変換確定を待たずに保存操作ができる

**関連要件**: REQ-IME-106

**詳細シナリオ**:
1. 日本語入力中（IME 変換候補表示中）
2. Cmd+S を押す
3. NTTextView.keyDown が `ntSaveDocument` 通知を発火
4. MainWindowView が受け取り `appState.syncActiveTab()` を呼び出す
5. その時点の `textView.string`（未確定テキスト含む可能性）で Notion 同期が実行される
6. エラーなしで完了（同期結果は別途ステータスバーで確認可能）

**前提条件**:
- Notion Integration Token が設定済み
- 保存先データベース・ページが選択済み

**優先度**: Must Have

---

### ストーリー 5: 変換確定後も自動保存が機能する 🔵

**信頼性**: 🔵 *scheduleAutoSave() L114-123・ユーザヒアリングより*

**私は** 自動保存に頼るユーザー **として**
**日本語変換を確定した後、3秒以内に自動的にローカル状態が保存されるようにしたい**
**そうすることで** 手動保存を意識せず日本語入力できる

**関連要件**: REQ-IME-002, REQ-IME-003, REQ-IME-202

**詳細シナリオ**:
1. IME 変換を確定（hasMarkedText() == false）
2. textDidChange が発火し `appState.updateContent()` が呼ばれる
3. `scheduleAutoSave()` が 3 秒のデバウンスをセット
4. 3 秒後、PersistenceManager.shared.saveTabs() でローカル保存
5. 追加入力があればデバウンスがリセットされ最後の入力から 3 秒後に保存

**優先度**: Must Have
