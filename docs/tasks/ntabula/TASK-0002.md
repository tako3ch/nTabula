## タスク概要

`Sources/App/AppState.swift` の初期化処理を修正し、Notion Integration Token を UserDefaults ではなく Keychain から読み書きするようにする。既存の `notionToken` 参照を PersistenceManager.loadToken() / saveToken() 経由に変更する。

## 依存タスク

- **前提タスク**: [TASK-0001: PersistenceManager Keychain メソッド実装](TASK-0001.md)
- **後続タスク**: [TASK-0004: NotionService URLSession注入対応](TASK-0004.md)

## 完了条件

- [ ] AppState.init() で `notionToken = persistenceManager.loadToken()` が呼ばれる
- [ ] `updateNotionToken(_ token: String)` が `persistenceManager.saveToken(token)` を呼ぶ
- [ ] UserDefaults への `notionToken` 直接書き込みが削除されている
- [ ] アプリ起動時に既存 UserDefaults トークンが自動的に Keychain に移行される（TASK-0001の loadToken 内で実施）

## 実装詳細

### 1. AppState.init() の修正 🔵

**信頼性**: 🔵 *dataflow.md フロー8・interfaces.swift AppStateより*

**実装ファイル**: `Sources/App/AppState.swift`

```swift
// 修正前
notionToken = defaults.string(forKey: "nTabula.notionToken") ?? ""

// 修正後
notionToken = persistenceManager.loadToken()
```

### 2. updateNotionToken の修正 🔵

**信頼性**: 🔵 *interfaces.swift AppState.updateNotionToken()より*

```swift
func updateNotionToken(_ token: String) {
    notionToken = token
    persistenceManager.saveToken(token)   // Keychain に保存
    // UserDefaults への直接書き込みを削除
    Task {
        notionService.updateToken(token)
        await fetchDatabases()
        await fetchPages()
    }
}
```

### 3. UserDefaults の旧 notionToken キー参照削除 🔵

**信頼性**: 🔵 *persistence-schema.md nTabula.notionToken 廃止より*

- `PersistenceManager` の `saveNotionToken` / `loadNotionToken` メソッドがあれば削除
- UserDefaults での "nTabula.notionToken" への直接アクセスをすべて除去
