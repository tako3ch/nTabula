## タスク概要

`Sources/Utilities/PersistenceManager.swift` に `init(defaults: UserDefaults = .standard)` overload を追加し、テスト時に独立した UserDefaults インスタンスを注入できるようにする。テスト時は `UserDefaults(suiteName: "test-\(UUID())")` を注入することで、本番データを汚染しない隔離されたストレージでテストを行える。

## 実装詳細

### 1. UserDefaults stored property の追加 🔵

**信頼性**: 🔵 *interfaces.swift PersistenceManager.init(defaults:)より*

**実装ファイル**: `Sources/Utilities/PersistenceManager.swift`

```swift
final class PersistenceManager {
    static let shared = PersistenceManager()

    private let defaults: UserDefaults  // 追加（既存の .standard 参照を置き換え）

    // 本番用 private init（シングルトン経由）
    private init() {
        self.defaults = .standard
    }

    // テスト用 init（NFR-302）— internal アクセス
    init(defaults: UserDefaults) {
        self.defaults = defaults
    }

    // ...
}
```

### 2. UserDefaults.standard 直接参照の置き換え 🔵

**信頼性**: 🔵 *NFR-302より*

全メソッド内の `UserDefaults.standard.xxx` を `defaults.xxx` に変更:

```swift
// 修正前
UserDefaults.standard.set(encoded, forKey: "nTabula.tabs")

// 修正後
defaults.set(encoded, forKey: "nTabula.tabs")
```

## 注意事項

- テスト用 `init(defaults:)` は `internal`（testable アクセスのため）にする
- `shared` の `private init()` は維持する（外部からの直接インスタンス生成を防ぐため）
- Keychain 部分（saveToken/loadToken）はデフォルトの Keychain を使用するため注入対象外
